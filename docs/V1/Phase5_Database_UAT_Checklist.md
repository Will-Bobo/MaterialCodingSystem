## 前置准备（建议）

1. 确认可以正常启动应用并进入主界面。
2. 准备以下文件/工具（按需）：
   - 一个**有效**的 `.db` 备份文件（可由“导出数据库”或“自动备份”生成）
   - 一个**损坏**的 `.db` 文件（可将有效 `.db` 复制后截断/写入随机内容）
   - 一个**非 `.db`** 文件（例如 `.txt`）
   - 可选：SQLite 查看工具（用于核对恢复后数据是否回到备份时刻）
3. 若需验证“文件被占用”场景：使用外部程序打开目标 `.db` 并保持占用（不关闭）。

| A-001 | 启动程序后是否自动生成备份 | 1）关闭应用；2）启动应用并进入主界面；3）等待启动完成（不做任何操作） | 启动后自动生成 1 份备份文件（无需用户交互） |  |  |

| C-002 | 是否仅允许 `.db` 文件 | 1）打开文件对话框；2）观察文件过滤器；3）尝试选择非 `.db` 文件 | 对话框默认过滤 `.db`；若选择非 `.db`，恢复前置校验应拒绝并提示 |  |  |

| C-006 | 重启后数据是否已恢复 | 1）在恢复前准备可辨识数据差异（如新增/废弃几条记录）；2）执行恢复并重启；3）验证数据 | 重启后数据状态与备份时刻一致（恢复=覆盖当前数据） |  |  |

# D. 异常场景

| 编号 | 测试项 | 操作步骤 | 预期结果 | 实际结果 | 是否通过 |
|---|---|---|---|---|---|
| D-001 | 选择损坏 `.db` 文件 | 1）选择一个损坏的 `.db`；2）执行恢复 | 恢复被拒绝并提示“源文件无效/不可用”；当前主库不被破坏 |  |  |
| D-002 | 选择非 `.db` 文件 | 1）在打开对话框中选中非 `.db` 文件（若可）；2）执行恢复 | 恢复被拒绝并提示“必须为 .db”；主库不被破坏 |  |  |

| D-005 | 无权限目录（导出/备份/恢复） | 1）选择无写权限目录进行“导出数据库”；2）选择无读权限文件进行“恢复数据库”（如可构造） | 对应操作失败可感知且不阻断主流程；恢复失败不破坏主库 |  |  |
