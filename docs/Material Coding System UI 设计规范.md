# Material Coding System UI 设计规范（V1.1）

## 版本说明

**当前版本：** V1.1

**本版本增强：**

- 决策可视化强化（候选高亮）
- 操作效率优化（固定主操作区）
- 可读性与可访问性增强
- UI 工程约束（资源、触发器）

## 1. 规范目标

本规范用于统一 Material Coding System（MCS）的界面设计与交互行为，确保：

- **一致性（Consistency）**
- **可读性（Readability）**
- **高效决策（Decision Efficiency）**
- **可扩展性（Scalability）**

## 2. 设计原则

### 2.1 核心原则

1. 查重优先（Search First）
2. 决策可视化（Decision Visible）
3. 输入最小化（Minimal Input）
4. 状态明确（State Explicit）

### 2.2 禁止事项

- ❌ 不允许页面纯表单堆叠（上下结构）
- ❌ 不允许多来源状态表达（多个提示冲突）
- ❌ 不允许默认系统风格（未统一样式）
- ❌ 不允许强烈视觉噪音（刺眼红色、大面积警告）

## 3. 视觉设计规范（Visual System）

### 3.1 配色规范（Color Palette）

| 类型        | 颜色     | 用途               |
| ----------- | -------- | ------------------ |
| 主色        | `#2563EB` | 主按钮 / 关键操作  |
| 主色 Hover  | `#1D4ED8` | Hover 状态         |
| 成功        | `#16A34A` | 成功提示           |
| 警告        | `#D97706` | 查重提示 / 决策    |
| 错误        | `#DC2626` | 校验错误           |
| 背景        | `#F3F3F3` | 页面背景           |
| 卡片        | `#FFFFFF` | 内容容器           |
| 边框        | `#E5E7EB` | 分割线             |

### 3.2 字体规范（Typography）

| 类型 | 字号   | 用途          |
| ---- | ------ | ------------- |
| 标题 | 15–16  | 模块标题      |
| 正文 | 12     | 输入 / 列表   |
| 辅助 | 11     | 提示 / Hint   |

**FontFamily：**

- Segoe UI
- Microsoft YaHei UI

### 3.3 圆角与阴影（Elevation）

- **CornerRadius：** 6px

**阴影：**

- BlurRadius：8–12
- Opacity：0.1–0.2

**用途：**

- 仅用于卡片（Card）
- 禁止用于按钮或文本

卡片阴影的具体数值须满足 **3.6 阴影使用限制（强制）**。

### 3.4 可读性约束（强制）

- 辅助文字颜色不得低于 `#6B7280`
- 禁止使用低对比度灰色（如 `#9CA3AF` 以下）
- 所有提示必须在普通显示器环境下可读

### 3.5 风险高亮规范（新增）

当系统识别为「高相似物料」时（如规则命中或高置信度），**必须**满足以下至少一种：

- 行背景：`#FEF2F2`（浅红）  
  **或**
- 边框：`#FCA5A5`

**禁止：**

- 仅使用文字颜色标识风险

**目的：** 强化「重复风险」的视觉感知，降低误创建概率。

### 3.6 阴影使用限制（强制）

- 仅允许用于卡片（Card）
- BlurRadius ≤ 10
- Opacity ≤ 0.15

**禁止：**

- 在按钮或文本上使用阴影

## 4. 布局规范（Layout System）

### 4.1 页面结构（强制）

**Split View（左右布局）**

| 区域 | 宽度        | 内容                   |
| ---- | ----------- | ---------------------- |
| 左侧 | 320–360px   | 输入区（Input Card）   |
| 右侧 | 自适应      | 候选区（Smart Panel）  |

### 4.2 布局示意

```
+-------------------+---------------------------+
| 输入卡片          | 候选物料（实时匹配）      |
|                   |                           |
| 分类              | 状态提示                  |
| Spec（核心）      | DataGrid / 列表           |
| 描述              |                           |
| 名称              |                           |
| 品牌              |                           |
|                   |                           |
| [创建按钮]        |                           |
+-------------------+---------------------------+
```

### 4.3 卡片规范（Card）

- 必须使用 Border + 阴影

**特征：**

- 白底
- 圆角
- 内边距 16px

### 4.4 候选面板尺寸约束

右侧候选区域必须满足：

- MinWidth ≥ 600px

**防止：**

- DataGrid 列挤压
- 信息不可读

### 4.5 操作区固定规则（强制）

主操作按钮（如「创建主物料」）必须：

- 固定在卡片底部（Sticky）
- 不随滚动消失

**目的：** 提升高频操作效率。

## 5. 组件规范（Component Rules）

### 5.1 输入控件（TextBox）

**间距：**

- Label 与输入：4px
- 输入之间：12px

**Spec 输入：**

- 必须为视觉主焦点

### 5.2 按钮规范（Button）

| 类型     | 样式     |
| -------- | -------- |
| 主按钮   | 蓝底白字 |
| 次按钮   | 灰边框   |
| 危险按钮 | 红色     |

### 5.3 DataGrid

**规则：**

- 禁止默认样式
- 必须去掉厚边框
- 行高统一
- 操作列统一右侧

### 5.4 状态提示规范（增强）

所有状态提示必须同时具备：

- 文本（必选）
- 颜色（必选）
- Icon（推荐）

**示例：**

- ✅ 未发现匹配
- ⚠ 检测到相似物料
- 🚫 存在冲突

**目的：** 提升可读性与色弱用户可用性。

## 6. 状态表达规范（State System）

### 6.1 单一状态源（强制）

所有查重状态统一来自：

`CreateMaterial.CandidateStatus`

### 6.2 状态映射

| 状态         | 表达                         |
| ------------ | ---------------------------- |
| Searching    | 「搜索中…」+ Loading         |
| HasCandidate | 警告（橙色）                 |
| NoCandidate  | 成功（绿色）                 |
| Idle         | 无提示                       |

### 6.3 可视规则

- ✔ 同一时间只能出现一种状态提示
- ❌ 禁止多 TextBlock 冲突

### 6.4 实现方式（强制）

所有状态展示必须通过：

- Binding + DataTrigger 实现

**禁止：**

- 在 code-behind 中手动控制 UI 可见性或颜色

## 7. 决策交互（Decision UX）

### 7.1 决策条（Decision Bar）

**触发条件：**

- 存在候选物料（HasCandidate）

### 7.2 样式规范

- 背景：`#FFF7ED`（浅橙）
- 边框：`#FDBA74`
- 圆角：6px

### 7.3 按钮规范

- `[作为替代料]` → 主按钮（蓝）
- `[仍然新建]` → 次按钮

### 7.4 决策条组件化（推荐）

当前 DecisionBar 建议升级为 InfoBar 模式。

**结构：**

`[Icon] 提示文本                [操作按钮]`

**样式：**

- 浅色背景（根据状态）
- 圆角
- 内边距统一

**目的：** 统一系统级提示交互。

## 8. 空状态（Empty State）

### 8.1 空状态（增强）

当系统处于 Idle 状态（未输入）时，右侧区域必须显示：

- 图标（如 🔍）
- **主提示：**「输入规格号开始查重」
- **次提示：**「支持规格号或描述模糊匹配」

**禁止：**

- 空白区域
- 仅显示空 DataGrid

## 9. Dialog 规范（CategoryDialog）

### 9.1 结构

卡片式布局（禁止裸 Grid）

### 9.2 要求

- ✔ 标题加粗
- ✔ 输入间距统一
- ✔ 按钮右对齐
- ✔ 错误提示使用红色（小字体）

## 10. 间距规范（Spacing）

| 项目         | 数值  |
| ------------ | ----- |
| 模块间距     | 16px  |
| 输入间距     | 12px  |
| 标签间距     | 4px   |
| 卡片内边距   | 16px  |

## 11. UI 工程约束（Engineering Rules）

### 11.1 资源管理（强制）

以下内容必须定义为 StaticResource：

- 颜色（Color / Brush）
- CornerRadius
- Button Style
- Card Style

**禁止：**

- 在 XAML 中硬编码颜色或样式

### 11.2 状态驱动 UI（强制）

UI 必须由 ViewModel 状态驱动：

- 使用 Binding
- 使用 DataTrigger

**禁止：**

- code-behind 控制 UI 状态

### 11.3 样式复用

所有页面必须复用统一样式资源：

- CardStyle
- PrimaryButton
- TextStyle

**禁止：**

- 页面内重复定义样式

## 12. 交互增强（UX Enhancement - 预留）

（当前版本可不实现，但需预留）

- ✔ Paste Intelligence（粘贴解析）
- ✔ 实时校验状态点（绿/黄/红）
- ✔ 搜索防抖（已实现）
- ✔ 键盘优先（Enter 创建）

## 13. 验收标准（Definition of Done）

1. 页面为左右布局（必须）
2. 所有输入在卡片中
3. 所有状态统一来源
4. 无默认 WPF 样式残留
5. 主按钮视觉统一
6. 决策路径清晰（查重 → 决策 → 创建）
7. ✔ 高相似物料必须视觉高亮
8. ✔ 主操作按钮始终可见
9. ✔ 页面无空白状态
10. ✔ 所有样式来源统一资源
11. ✔ 无 code-behind UI 控制
